# PM Assistant —— 产品经理的 AI 外脑

> 产品助手不是预设模板和静态文档的拼凑，而是一个**工作流与知识库动态耦合的进化系统**。你运行的每一次工作流——写 PRD、出方案、做复盘——都在实时喂养知识库；而知识库中积累的案例、决策模式和领域约束，又反过来驱动下一次工作流的路径选择。知识库不是存档柜，而是工作流的引擎；工作流不是机械流程，而是知识生长的载体。两者互为因果，螺旋上升。它不是替你工作的工具，是外置一个持续校准你判断方式的工作大脑。

## 这是什么？

**产品助手（PM Assistant）** 不是预设模板和静态文档的拼凑，而是一个**工作流与知识库动态耦合的进化系统**。你运行的每一次工作流——写 PRD、出方案、做复盘——都在实时喂养知识库；而知识库中积累的案例、决策模式和领域约束，又反过来驱动下一次工作流的路径选择。知识库不是存档柜，而是工作流的引擎；工作流不是机械流程，而是知识生长的载体。两者互为因果，螺旋上升。你投入的项目越多，它对你的领域理解越深，输出的精准度和匹配度就越高。本质上，这是一套**越用越聪明的认知飞轮**。

它是产品经理的 AI 工作搭档，覆盖 PRD 撰写、方案设计、原型设计规格三大核心工作流，内置案例库、产品模式笔记、错误避坑清单和设计检查清单。B 端和政务数据产品为特长领域，C 端产品场景同样适用。

它和 [外脑（Exobrain）](https://github.com/wsdlp46/exobrain) 同属 WorkBuddy AI 协作体系，定位互补：

| 维度 | 产品助手（PM Assistant） | 外脑（Exobrain） |
|------|------------------------|-----------------|
| **用户** | 产品经理 | 所有知识工作者 |
| **角色** | 产品经理的 AI 外脑，内置专业工作流 | 零预设通用搭档，从材料中学习 |
| **工作流** | PRD/方案/原型三大专业流程 | 搭结构、入库、加载上下文、复盘 |
| **上手方式** | 配置工作区，直接开工 | 先扔材料，再搭结构 |
| **内置知识** | 产品模式笔记、案例库、避坑清单、检查清单 | 无 |

## 快速开始

### 安装

在 WorkBuddy 桌面客户端中，左侧边栏「专家」→ 右上角「+」→「从本地文件导入」→ 导入本项目目录。

### 第一步：准备工作区

你需要一个本地目录作为工作区根目录。两种方式：

**方式 A：自己建（推荐）**

按下方「工作区目录结构」创建目录，把项目材料放进去。至少准备：
- 一个 `01通用/` 目录，里面放模板、检查清单、设计规范
- 一个 `03知识库/工作案例库/` 目录，放过往项目案例
- 一个 `02当前项目/项目名/` 目录，放当前项目的材料

**方式 B：让产品助手帮你建**

对产品助手说：

> 帮我搭一套产品知识库结构，我的工作区根目录是 `/Users/xxx/工作区`

产品助手会创建标准目录结构，并生成 `00主页.md` 作为入口。

### 第二步：放入项目材料

把当前项目的材料放进工作区：

- 会议纪要、需求输入、客户反馈 → `02当前项目/项目名/01-项目输入/`
- 标书、功能清单、竞品资料 → `02当前项目/项目名/01-项目输入/`
- 过往同类项目的 PRD、方案 → `03知识库/工作案例库/`

**关键**：`01通用/` 里的模板和检查清单是产品助手产出质量的基准，建议先放入。模板和检查清单没有的话，产品助手也能工作，但输出质量会打折扣。

### 第三步：开始对话

打开工作区后，直接说你要干什么：

> 帮我写 [项目名] 的 PRD，工作区在 `/Users/xxx/工作区/02当前项目/项目名`

产品助手会自动加载工作区的上下文（关键决策、会议纪要、案例库），然后按 PRD 工作流执行。

## 典型对话示例

以下是 5 个常用场景的具体话术：

### 1. 写 PRD

> 帮我写双碳平台统一服务输出系统的 PRD，工作区在 `/Users/xxx/工作区/02当前项目/双碳平台`

产品助手会：加载上下文 → 查案例库匹配同类项目 → 按模式笔记判断适用模式 → 套 PRD 模板 → 跑检查清单 → 输出完整 PRD 文档。

### 2. 出方案

> 设计一套多租户权限方案，这个项目是 SaaS 平台，需要支持租户隔离和贴牌部署。工作区在 `/Users/xxx/工作区/02当前项目/xxx平台`

产品助手会：查案例库中 SaaS 多租户相关案例 → 按论证链组织 → 输出关键决策表 → 跑设计检查清单。

### 3. 做原型规格

> 按端 → 模块 → 页面的结构，给这个项目出原型设计规格。PRD 已确认，设计规范在 `01通用/设计库/`。工作区在 `/Users/xxx/工作区/02当前项目/xxx平台`

产品助手会：加载 `proto-spec-generator` Skill → 按四级结构逐页生成 → 每页等确认后再进下一页。

### 4. 入库新材料

> 入库这个文件：`/Users/xxx/工作区/02当前项目/xxx平台/01-项目输入/客户会议纪要_2026-06-15.md`

产品助手会：判断类型 → 提取关键决策 → 更新会议时间线 → 更新 `_index.md` 和 `关键决策.md`。

### 5. 项目复盘

> 复盘这个项目，更新知识库。工作区在 `/Users/xxx/工作区/02当前项目/xxx平台`

产品助手会：扫描全过程 → 归因分析 → 修复知识库 → 沉淀案例 → 更新 MEMORY.md。

## 功能

- **PRD 撰写**：按标准模板产出完整需求文档，角色分析端侧前置、业务流程角色泳道图、功能描述精炼、Out of Scope 必含
- **方案设计**：按"场景——痛点——方案概述——关键决策表——排期"论证链组织输出
- **原型设计规格**：按端、模块、页面四级结构输出标准化设计规格文档
- **HTML 原型设计**：根据设计规格文档，依次分页设计，做好页面间跳转效果
- **案例库联动**：自动匹配过往项目案例，复用模式笔记、规避已知错误
- **知识库接入**：对接设计规范、检查清单、术语表等通用资产

## 工作区目录结构

```
工作区根目录/                     ← 用户指定
├── 00主页.md                    ← 工作区入口
├── 01通用/                      ← 跨项目共享资产（用户自备）
│   ├── 项目工作流规范.md
│   ├── 设计库/
│   ├── 模板/
│   └── 检查清单/
├── 02当前项目/                  ← 进行中的项目
│   └── 项目名/
│       ├── _index.md            ← 项目索引
│       ├── 00-项目总览.md
│       ├── 01-项目输入/         ← 会议纪要、需求输入、标书
│       ├── 02-需求文档/         ← PRD 等产出物
│       ├── 03-原型设计/         ← 原型规格、HTML 原型
│       ├── 04-架构图/
│       ├── 05-评审纪要/
│       ├── 06-对外材料/
│       └── _source/             ← 二进制原始文件（PDF、Word、Excel 等）
├── 03知识库/                    ← 案例库、方法论文档（用户自备）
│   └── 工作案例库/
│       ├── 案例库主页.md
│       ├── 方法论_模式笔记.md
│       └── 方法论_错误笔记.md
└── 04历史项目/                  ← 已完成项目归档
```

> **注意**：`_source/` 里的二进制文件（PDF、Word、Excel）不会直接被产品助手读取。建议先转换为 Markdown 再放入，或者让产品助手通过 `project-intake-pm` 自动转换。

## 自备资产清单

以下文件需要你自己准备（放进工作区），产品助手不会自动生成：

| 文件 | 位置 | 作用 | 没有会怎样 |
|------|------|------|-----------|
| PRD 模板 | `01通用/模板/PRD模板.md` | PRD 撰写的骨架 | 产品助手会按默认格式写，但可能不符合你的公司规范 |
| 需求评审检查清单 | `01通用/检查清单/需求评审检查清单.md` | PRD 写完后逐项自查 | 不跑检查清单，遗漏风险增加 |
| 设计检查清单 | `01通用/检查清单/设计检查清单.md` | 方案设计时自查 | 同上 |
| 案例库主页 | `03知识库/工作案例库/案例库主页.md` | 案例库索引和领域标签 | 无法自动匹配过往案例 |
| 模式笔记 | `03知识库/工作案例库/方法论_模式笔记.md` | 产品模式速查（招投标驱动、平台型、SaaS 多租户等） | 无法自动判断适用模式 |
| 错误笔记 | `03知识库/工作案例库/方法论_错误笔记.md` | 已知错误模式和避坑记录 | 无法规避已知错误 |
| 设计规范 | `01通用/设计库/` | 原型设计时的视觉和交互规范 | 原型规格可能不符合你的设计标准 |

**建议**：一开始至少准备 PRD 模板和检查清单，案例库可以边用边积累。

## 依赖 Skill

| Skill | 关系 | 说明 | 是否需要额外安装 |
|-------|------|------|----------------|
| pm-workflow | 内置 | 核心工作流引擎（PRD/方案/原型），封装方法论、案例库、检查清单 | 否，已随产品助手导入 |
| project-intake-pm | 外部 | 项目材料入库自动整理（PM 场景） | 是，需要单独安装到 WorkBuddy |
| project-retrospective-pm | 外部 | 项目复盘驱动知识库迭代（PM 场景） | 是，需要单独安装到 WorkBuddy |
| proto-spec-generator | 外部 | 原型设计规格文档生成 | 是，需要单独安装到 WorkBuddy |
| prototype-html-pin | 外部 | HTML 原型生成 | 是，需要单独安装到 WorkBuddy |

**安装方式**：在 WorkBuddy 中「技能」→「从本地文件导入」→ 分别导入对应 Skill 目录。
