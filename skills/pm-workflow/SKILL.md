# PM Workflow — 产品经理工作流

本 Skill 封装产品经理三大核心工作流：PRD 撰写、方案设计、原型设计。每条工作流遵循"加载上下文 → 查案例 → 查模式 → 避坑 → 套模板 → 跑清单"的标准路径。

> 所有路径均为相对于用户配置的工作区根目录（`<workspace>/`）。

---

## 步骤0：上下文加载（所有工作流共用）

在执行任何工作流之前，强制加载当前项目的上下文：

1. 读取项目 `_index.md`（了解当前项目文件全貌）
2. 读取 `01-项目输入/关键决策.md`（确认所有前置设计约束）
3. 读取 `01-项目输入/会议纪要_时间线.md`（了解最新外部输入）
4. 读取 `01通用/项目工作流规范.md`（确认节点操作规则）
5. 打开 `03知识库/工作案例库/案例库主页.md`（准备案例匹配）

---

## 工作流 A：PRD 撰写

步骤0（上下文加载）已在上方执行。以下为 PRD 专属步骤：

### A1. 锁决定策约束

从上下文加载结果中提取本项目的关键决策，标注到 PRD 中：
- 每个功能模块标注依据：`决策依据：[[01-项目输入/关键决策#DXXX]]`
- 功能边界严格按决策约束，不得超出

### A2. 查模式笔记

打开 `03知识库/工作案例库/方法论_模式笔记.md`，判断当前产品阶段应适用的模式：

- **模式 1：招投标驱动** — 严格以 Excel 功能清单为准，按标书章节组织 PRD。详见 `01通用/方法论/招投标驱动原型设计方法论.md`（S1-S6 六阶段流水线）
- **模式 2：平台型产品** — 先定义角色和端侧，再展开功能模块
- **模式 3：SaaS 多租户** — 租户隔离、权限体系、贴牌部署为前置设计
- **模式 4：数据资产管理** — 按台账、确权、授权、评估、收益、流通、处置全链路
- **模式 5：从 0 到 1** — 用 BRD 框架，先发散再收敛

### A3. 查错误笔记

打开 `03知识库/工作案例库/方法论_错误笔记.md`，确认是否命中已知错误模式。案例库领域的匹配已在步骤0完成。

### A4. 套模板

以 `01通用/模板/PRD模板.md` 为骨架，参考步骤0匹配到的深度案例的章节组织方式。

### A5. 跑检查清单

写完后对照 `01通用/检查清单/需求评审检查清单.md` 逐项自查。

## 工作流 B：方案设计

步骤0（上下文加载）已在上方执行。

### B1. 查案例库关键决策

从步骤0匹配到的同类案例中，重点查看关键决策点，特别是竞品对比和技术选型。

### B2. 按论证链组织

```
场景描述 → 痛点分析 → 方案概述 → 关键决策表 → 报价/排期
```

### B3. 关键决策表模板

| # | 决策点 | 备选方案 | 选择理由 | 影响评估 |
|---|--------|---------|---------|---------|
| 1 | {决策描述} | A: {方案A} / B: {方案B} | {为什么选这个} | {对成本/时间/架构的影响} |

### B4. 查错误笔记 + 跑设计检查清单

对照 `01通用/检查清单/设计检查清单.md`。

---

## 工作流 C：原型设计

步骤0（上下文加载）已在上方执行。上下文加载中包含关键决策.md，因此原型设计时已知所有设计约束。

### C1. 前置确认

确认 PRD 和架构图是否已定稿。未定稿则先补前序交付物。

### C2. 查设计规范

参考 `01通用/设计库/` 下的设计规范文件和 B 端原型 6 条准则（详见 `references/b-prototype-rules.md`）。

### C3. 查同类原型

在案例库中找同领域已有原型作为参考。

### C4. 加载原型设计规格生成器

当需要按页面输出设计规格时，加载 `proto-spec-generator` Skill，按以下流程执行：

1. **前置确认**：完成端侧确认、模块对齐、业务流程、页面划分、已有能力复用、产出架构清单
2. **逐页生成**：每页严格按模板规范输出（功能概述、权限说明、页面结构、页面内容、交互规则）
3. **门控确认**：每页输出后等用户确认，确认通过后再进入下一页
4. **关键约束**：不自创功能、不自加字段、命名对齐原文、优先简单交互

## 通用规则

- **输出语言**：中文专业沟通
- **功能描述**：阿拉伯数字分点，单条不超过 120 字
- **平台类产品**：先定义角色和端侧，再展开功能
- **招投标项目**：严格以 Excel 功能清单为准
- **交付标准**：可直接向客户展示的水平
- **产品原型准则**：详见 `references/b-prototype-rules.md`（B 端原型 6 条准则为核心参考，C 端场景可灵活适配）
- **完成后**：更新项目 `_index.md` 最近更新条目；阶段性完成时提醒用户是否执行 `project-retrospective-pm` 复盘
